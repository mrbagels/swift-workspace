Workspace Engine Split

swift-workspace separates shared application mechanics from platform presentation and optional adapters. The Mac shell is one renderer over the engine, not the engine itself.

Open Markdown Download Markdown

Core Boundary

WorkspaceCore owns pure Swift concepts: typed commands, command roles, command reference grouping, typed route descriptors, metadata patches, availability, command search, command policy, route-open requests, URL parsing, scene requests, scene values, scene collections, and shared restoration payloads.

It must not import SwiftUI, TCA, SQLiteData, CloudKit, AppKit, or UIKit.

Reducer Boundary

WorkspaceTCA owns platform-neutral transitions: route selection, command execution, palette state, policy enforcement, scene-request delegates, restoration, recents, collapsed sections, navigation replacement, and route metadata reconciliation.

It must not own Mac chrome, iOS navigation presentation, storage writes, CloudKit writes, server calls, or app domain behavior.

Renderer Boundary

MacWorkspaceShell and IOSWorkspaceShell render the same engine with native platform choices. Shared behavior moves down. Platform chrome stays in the renderer.

Adapter Boundary

Persistence and external services are optional products. UserDefaults, SQLiteData, CloudKit, and future server clients should be imported only by apps that need them.

Server Boundary

The server is a companion service for accounts, entitlements, templates, jobs, integrations, diagnostics, and support workflows. The optional testing product owns cassettes, replay, and contracts. iCloud and local storage remain primary for user-owned data.
