Phased Implementation Plan

A staged path from clean scaffold to professional reusable workspace product, using the old Mac Shell as prototype reference while building the new source of truth in swift-workspace.

Open Markdown Download Markdown

Phase 0: Fresh Root And Guardrails

Complete for the initial scaffold: new root, package, XcodeGen project, docs lifecycle, first products, demo apps, and verification scripts.

Phase 1: Stabilize WorkspaceCore

In progress: command roles and grouping, route metadata patches, URL route parsing, Codable storage-safe models, and expanded core tests are in place.

Phase 2: Make WorkspaceTCA Boring

In progress: reducer tests now cover palette lifecycle, recents, policy, route-open behavior, scene delegates, restoration, and metadata reconciliation.

Phase 3: Port Proven Mac Shell Behavior

In progress: Mac shell configuration, native split rendering, command palette UI, command reference UI, command-menu entry points, and Mac-specific restoration are in place.

Phase 4: Build The iOS And iPadOS Renderer

In progress: iOS shell configuration, restoration, command-search sheet, badge preferences, scene-aware route actions, and demo adoption are in place.

Phase 5: Persistence And iCloud Primary Storage

In progress: file-backed restoration storage, CloudKit record names, zones, conflict policy, restoration envelopes, route metadata envelopes, and adapter contracts are in place.

Phase 6: Companion Server

Add optional server capabilities for entitlements, templates, jobs, diagnostics, and integrations without replacing iCloud.

Phase 7: Distribution And Adoption

Publish from the root Swift package with DocC, quickstarts, release checks, and starter apps.

Phase 8: Professional Polish

Run design, accessibility, performance, API, and demo polish passes.
